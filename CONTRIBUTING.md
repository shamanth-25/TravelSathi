# Contributing to TravelSathi

Thank you for your interest in contributing to TravelSathi!

## How to Contribute

### 1. Fork and Clone

Fork the repository and clone it locally:

```bash
git clone <repository-url>
cd TravelSathi
```

### 2. Create a Branch

Use descriptive branch names:

```bash
git checkout -b feature/your-feature-name
```

Examples:

* feature/frontend-enhancement
* feature/backend-improvement
* bugfix/ollama-issue

### 3. Development Setup

Create a virtual environment:

```bash
python -m venv venv
source venv/bin/activate
```

Install dependencies:

```bash
pip install -r requirements.txt
```

### 4. Coding Guidelines

* Follow PEP 8 standards.
* Write meaningful commit messages.
* Keep functions modular and reusable.
* Add comments for complex logic.
* Update documentation when introducing new features.

### 5. Testing

Before submitting changes:

```bash
pytest
```

Ensure:

* Existing functionality works correctly.
* No tests are broken.
* New features include appropriate tests.

### 6. Pull Requests

Before opening a pull request:

* Sync with the latest main branch.
* Resolve merge conflicts.
* Ensure code quality checks pass.
* Provide a clear description of the changes.

### 7. Reporting Issues

When reporting bugs, include:

* Steps to reproduce
* Expected behavior
* Actual behavior
* Screenshots (if applicable)
* Environment details

## Code of Conduct

By contributing to TravelSathi, you agree to maintain a respectful and collaborative environment.

Thank you for helping improve TravelSathi!
